Construção de servidor em Node.JS usando Express para gestão de arquivos sonoros
Supervisor José Carlos Ramalho
Data Início 26-10-2019
Data Final 28-10-2019
  • A81952 - Mateus Ferreira da Silva - a81952@alunos.uminho.pt

    • Abstract:

    Este trabalho baseou-se na contrução de um servidor em Node.JS capaz de manter e manipular um arquivo sonoro. Para isso, o dataset dos arquivos sonoros é mantido no servidor sob a forma de um ficheiro em formato JSON, permitindo a sua consulta e uma filosofia CRUD, isto é, é permitido adicionar novos arquivos sonoros, atualizar arquivos sonoros já existentes, eliminar arquivos sonoros e ainda consultar arquivos sonoros já existentes.

    Para ter acesso a todos os arquivos sonoros basta aceder à página inicial, onde se podem encontrar estes os registos existentes até ao momento.

    Todas as outras opções surgem como funcionalidades implementadas de forma simples e intuitiva, usando para isso botões e formulários.

    Deliverables:

    1. PR.xml - XML do Project Record
    2. PR.html - HTML do Project Record
    3. index.js - tratamento dos pedidos que chegam ao servidor
    4. layout.pug - layout básico da aplicação
    5. erro.pug - página de erro da aplicação
    6. index.pug - página inicial da aplicação
    7. arquivo_singular.pug - página de consulta de um arquivo sonoro
    8. update.pug - página de atualização de um arquivo sonoro
    9. add.pug - página de adição de um arquivo sonoro
    10. arq-son-EVO.json - ficheiro que serve de base de dados
    11. update.js - função de atualização de um registo
    12. delete.js - função de eliminação de um registo
    13. www - ficheiro para executar o servidor
